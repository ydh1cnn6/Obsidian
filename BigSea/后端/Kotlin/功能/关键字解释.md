# expect & actual
## expect
主要作用
声明平台无关的API：在 common 模块中声明平台无关的函数、类、属性等接口
定义多平台共享契约：为不同平台提供统一的接口定义，确保各平台实现的一致性
实现平台特定功能：允许在共享代码中调用平台特定的功能，如文件操作、网络请求等
## actual
实现预期声明：为在 common 模块中使用 expect 关键字声明的函数、类、属性等提供平台特定的具体实现
平台差异化实现：允许不同平台（如 JVM、JS、Native）根据自身特性提供不同的实现方式
保持接口一致性：确保各个平台的实现遵循相同的接口约定

## 使用方式
1、在 common 模块中声明
```kotlin
 expect fun platformName(): String
   expect class PlatformService() {
       fun doSomething()
   }
```
2、在各平台模块中实现
```kotlin
   // JVM平台
   actual fun platformName(): String = "JVM"
   actual class PlatformService {
       actual fun doSomething() { /* JVM实现 */ }
   }
   
   // JS平台  
   actual fun platformName(): String = "JS"
   actual class PlatformService {
       actual fun doSomething() { /* JS实现 */ }
   }
   
```
